# 開発（Engineering）

技術設計・アーキテクチャ・実装ドキュメントを管理します。

## サブフォルダ
- `docs/` - 技術ドキュメント・設計書
- `debug-log/` - デバッグ・バグ調査ログ

## 技術スタック（確定）

| 領域 | 技術 | 理由 |
|------|------|------|
| フロントエンド | **Next.js (App Router) + TypeScript** | PWA対応・Vercel自動デプロイ・初心者でも学習資料豊富 |
| スタイリング | Tailwind CSS | 高速UI構築 |
| アプリ形式 | **PWA**（ネイティブアプリ不要） | App Store審査不要・カメラ/通知/決済すべて対応可 |
| デプロイ | **Vercel** | git pushで自動デプロイ・無料枠で本番運用可 |
| DB・認証・ストレージ | **Supabase** | 認証+DB+画像保存を一括管理・バックエンド不要 |
| 決済 | **Stripe** | 日本円対応・返金API充実・Webhook処理もシンプル |
| プッシュ通知 | **Web Push API + Service Worker** | PWAで対応可能・FCM不要 |
| 管理画面 | **Supabase Studio**（MVP初期） | 追加ツール不要 |

### Flutterをやめた理由
- Next.js + PWAでカメラ・通知・決済・グループ機能がすべて実装可能
- App Store審査（数週間）を経ずにgit push即公開できる
- 初心者にとってFlutter+バックエンドの2系統より1系統の方が圧倒的に学習コストが低い
- Vercelの自動デプロイで開発→公開のサイクルが劇的に速くなる

## 重要な実装制約

- 決済情報は直接保持しない（PCI DSS準拠の決済代行に委託）
- 参加費と手数料は別建てで明示する
- 証拠写真のメタデータ（EXIF・タイムスタンプ）は検証ロジックに使う
- 未達者の参加費を達成者に分配するロジックは実装しない

## MVP実装スコープ

### 含めるもの
- ユーザー認証（メール・Google・Apple）
- チャレンジ一覧・詳細表示
- 参加申込み・決済（Stripe）
- 証拠写真投稿（アプリ内カメラ）
- グループタイムライン
- 達成率計算
- 返金処理
- プッシュ通知（チェックイン催促）

### 含めないもの（後フェーズ）
- スポンサー向け管理画面
- 独自チャレンジ作成機能
- MVP選定機能
- Web対応
