## 確定技術スタック

### フロントエンド + アプリ（一体）
| 技術 | 理由 |
|------|------|
| **Next.js (App Router) + TypeScript** | PWA・SSR・API Routes・Vercel自動デプロイがすべて揃う |
| **Tailwind CSS** | 高速UI構築 |
| **PWA**（manifest + Service Worker） | App Store不要・カメラ/通知/決済すべて対応可・git push即公開 |

### バックエンド（Supabaseで代替）
| 技術 | 理由 |
|------|------|
| **Supabase** | PostgreSQL + Auth + Storage + Realtime を一括管理。バックエンドを別途構築不要。 |
| **Stripe** | 参加費受付・返金処理・Webhook |
| **Web Push API** | PWAプッシュ通知（FCM不要） |

### デプロイ
| 技術 | 理由 |
|------|------|
| **Vercel** | git push → 自動デプロイ。無料枠で本番運用可。ビルドログもClaude Codeに読ませてデバッグできる。 |

> **Flutterをやめた理由**: PWAでカメラ・通知・決済・グループ機能がすべて実装可能。App Store審査なしでgit push即公開。初心者には1系統の学習コストが圧倒的に低い。

## リスクと対策

| リスク | 対策 |
|--------|------|
| 法律面でNGが出る | Phase 0で弁護士確認 → 設計変更の余地を残す |
| 不正投稿（同じ写真の使い回し） | 画像ハッシュ検証・投稿時刻検証 |
| 返金への不満・クレーム | ルールの透明な明示・サポート体制の整備 |
| スポンサーが集まらない | MVP段階では運営提供の特典で代替（次回参加割引など） |
| ユーザー獲得コスト高 | 友達招待機能・SNSシェア設計を早期に実装 |
| グループの熱量が下がる | 8人以下の少人数設計・毎日のリマインド通知 |
