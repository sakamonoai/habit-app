| ユーザーにとって | スポンサーにとって |

## 仕組み図

```
┌─────────────────────────────────────────────────────┐
│                    チャレンジ参加                      │
│  参加費 1,000円 + システム利用料 100円                  │
└──────────────┬──────────────────────────────────────┘
               │
       毎日 証拠投稿（写真）
               │
       ┌───────┴────────┐
       ↓                ↓
   85%以上達成         85%未満
   参加費 返金          参加費 没収
                        → 運営収益
       ↓
   100%達成
   スポンサー景品・特典
```

## 返金・報酬の設計

| 達成率 | 結果 |
|--------|------|
| 100% | 参加費全額返金 + スポンサー景品（上位特典） |
| 85〜99% | 参加費全額返金 + 完走バッジ |
| 84%以下 | 参加費は返金なし |

**手数料の取り方（2パターン、法務確認後に確定）:**
- パターンA: システム利用料（100円など）を参加時に別建てで徴収
- パターンB: 達成者にも一部手数料控除（ただしUI上「全額返金」と誤解されやすいので要注意）

**推奨: パターンA**（ユーザーの納得感が高い）

## スポンサーモデル

### 協賛パターン
| パターン | 内容 |
|--------|------|
| 景品協賛 | 既存チャレンジに景品を提供する |
| テーマ協賛 | 既存チャレンジに社名・ブランドを乗せる |
| 独自チャレンジ立ち上げ | 自社商品を使うチャレンジを作る |

### 独自チャレンジの強み（スポンサー向けセールスポイント）
- 「広告を見てもらう」ではなく「商品を毎日使ってもらう」
- 継続利用の証拠写真がユーザーから集まる
- 完走者 = 商品の真のファン層にリーチできる
- 完走者にクーポンを渡せば購入転換につながる

### スポンサー課金モデル（案）
| プラン | 内容 | 費用感（参考） |
|--------|------|------------|
| 景品協賛 | 景品現物を提供（費用は現物コストのみ） | 景品費用のみ |
| テーマ協賛 | チャレンジにブランドを乗せる | 月額固定 |
| 独自チャレンジ | チャレンジ開設 + レポート提供 | 初期費 + 成果報酬 |
