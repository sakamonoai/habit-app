# 要件定義書 — 日本版チャレンジャーズ型習慣化アプリ

> 最終更新: 2026-03-21
> ステータス: ドラフト（弁護士確認前）

---

## 1. プロダクト概要

### コンセプト
「固定デポジット × 証拠提出 × 少人数グループ × スポンサー報酬」による
日本向け習慣化コミュニティアプリ。

韓国の챌린저스をベースに、**日本の法律・ユーザー心理に合わせて変形**したモデル。

### 一言説明
> 仲間と一緒にチャレンジして、達成したらお金が戻ってくる。スポンサーからのご褒美もある。

---

## 2. 事業モデル

### お金の流れ（法律安全設計）

```
ユーザー → 参加費（例: 1,000円）+ システム利用料（例: 100円）を支払う
                │
         ┌──────┴──────┐
         ↓              ↓
   85%以上達成        85%未満
   → 参加費を返金    → 参加費は返金なし（運営収益）
         ↓
   100%達成
   → スポンサーからの景品・特典を付与
```

**重要な原則（法律面）:**
- 未達者のお金を達成者に山分けしない
- 報酬原資はスポンサー提供のみ
- 参加費と手数料は参加前に明示
- 「稼げる」「儲かる」を訴求しない

### 収益源

| 収益源 | 内容 |
|--------|------|
| システム利用料 | チャレンジ参加時に固定徴収 |
| 未達成参加費 | 返金されなかった参加費（運営収益） |
| スポンサー掲載料 | スポンサーがチャレンジに協賛する費用 |
| 協賛チャレンジ費 | スポンサーが独自チャレンジを立ち上げる費用 |
| コマース送客手数料 | 完走者クーポン経由の購入に対する手数料 |

## 3. ユーザー種別

| 種別 | 説明 |
|------|------|
| 一般ユーザー | チャレンジに参加するメインユーザー |
| スポンサー企業 | チャレンジに協賛 or 独自チャレンジを立ち上げる企業 |

## 4. 機能要件

### 4-1. 一般ユーザー機能

#### 認証・アカウント
- [ ] メールアドレス + パスワードで登録
- [ ] Googleログイン / Appleログイン
- [ ] プロフィール設定（ニックネーム・アイコン・一言紹介）
- [ ] 習慣カテゴリの興味設定（初回onboarding）

#### チャレンジ一覧・検索
- [ ] チャレンジ一覧の表示（カテゴリフィルター・期間フィルター）
- [ ] チャレンジ詳細の閲覧（ルール・参加費・報酬・期間・参加者数）
- [ ] スポンサー協賛チャレンジの表示（バッジで識別）
- [ ] おすすめチャレンジのレコメンド

#### チャレンジ参加
- [ ] チャレンジへの参加申込み
- [ ] 参加費 + システム利用料の決済（クレジットカード / その他）
- [ ] グループへの自動割り当て（3〜8人）または友達グループ作成
- [ ] 参加時にルール・返金条件への同意
- [ ] 参加確認通知（メール + プッシュ）

#### 証拠提出（チェックイン）
- [ ] アプリ内カメラで写真撮影・投稿
- [ ] テキストコメントの追加
- [ ] 投稿可能時間帯の制限（チャレンジごとに設定）
- [ ] 1日の投稿締め切り管理
- [ ] 投稿履歴の確認

#### グループ・コミュニティ
- [ ] グループタイムライン（メンバーの投稿一覧）
- [ ] スタンプ・応援リアクション（いいね・ガッツポーズなど）
- [ ] コメント機能
- [ ] グループメンバーの達成率表示
- [ ] チーム全体の達成率サマリー

#### 個人ダッシュボード
- [ ] 参加中チャレンジの達成率・残日数
- [ ] 連続達成日数（ストリーク）
- [ ] 過去チャレンジ履歴・達成率
- [ ] 獲得バッジ・特典一覧

#### 返金・報酬
- [ ] チャレンジ終了後の達成率確認
- [ ] 85%以上達成時：参加費の返金処理（申請 or 自動）
- [ ] 100%達成時：スポンサー景品・クーポンの受取
- [ ] MVP受賞通知・景品発送対応

#### 通知
- [ ] 毎日のチェックイン催促通知
- [ ] 締め切り前リマインド通知
- [ ] グループメンバーの投稿通知
- [ ] チャレンジ終了・結果通知
- [ ] 返金・景品通知

## 5. 非機能要件

### パフォーマンス
- 画面読み込み: 3秒以内（通常環境）
- 画像アップロード: 10秒以内（5MB以下の写真）
- 同時接続: 初期1,000ユーザー想定、スケーラブルな設計

### セキュリティ
- 決済情報はPCI DSS準拠の決済代行サービスに委託（直接保持しない）
- 個人情報の暗号化保管
- 通信はHTTPS必須
- 不正ログイン対策（レートリミット・二要素認証オプション）

### 可用性
- 稼働率 99%以上（月次）
- メンテナンス時の事前通知

### プラットフォーム
- iOS（iPhone対応）
- Android
- Webは将来対応（MVP時点では不要）

## 6. チャレンジ設計ルール

### 基本パラメータ
| パラメータ | 仕様 |
|-----------|------|
| 期間 | 7日 / 14日 / 21日（固定選択肢。オーソリ保持期限のため最大21日） |
| 参加費 | チャレンジごとに固定（500〜3,000円程度） |
| システム利用料 | 参加費とは別建てで明示 |
| グループ人数 | 3〜8人 |
| 証拠投稿 | 1日1回（アプリ内カメラ必須） |
| 返金ライン | 85%以上の達成率 |
| 報酬ライン | 100%達成でスポンサー特典 |

### 達成率計算
```
達成率 = 実際に投稿した日数 ÷ チャレンジ期間の日数 × 100
```

### MVP選定（将来機能）
以下の基準で算出し、事前にユーザーへ公開：
- 達成率: 50%
- 継続日数: 20%
- コミュニティ貢献（応援・コメント数）: 20%
- 他ユーザー投票: 10%

## 7. 法律面の考慮事項（弁護士確認必須）

### 確認が必要な法律
| 法律 | 論点 |
|------|------|
| 賭博罪（刑法） | 参加費デポジット構造が賭博に該当しないか |
| 景品表示法 | スポンサー景品の上限・告知義務 |
| 資金決済法 | 参加費の預かり方が「前払式支払手段」や「資金移動」に該当しないか |
| 消費者契約法 | 返金条件の明示・不当条項に該当しないか |
| 特定商取引法 | アプリ内課金・返金ポリシーの表示義務 |
| 個人情報保護法 | 写真・行動データの取扱い・スポンサーへのデータ提供 |

### 安全サイドの設計原則
- ✅ 未達者のお金を達成者に分配しない
- ✅ 報酬原資はスポンサー提供のみ
- ✅ 参加前にルール・返金条件を明示・同意取得
- ✅ 「稼げる」「増える」の訴求をしない
- ✅ 高額参加費を避ける（上限設定）
- ❌ 参加者間での金銭山分け
- ❌ 運要素のある賞金設計（抽選は景表法確認要）
- ❌ 射幸心を煽るUI・訴求

## 8. MVP（最小検証版）のスコープ

### MVPに含めるもの
- ユーザー登録・ログイン
- チャレンジ一覧・参加
- 決済（クレジットカード）
- 毎日の証拠投稿（写真）
- グループタイムライン（投稿閲覧・リアクション）
- 達成率計算・返金処理
- プッシュ通知（チェックイン催促）

### MVPに含めないもの（後フェーズ）
- スポンサー向け管理画面
- 独自チャレンジ作成（スポンサー）
- MVP選定機能
- 詳細分析・レポート
- Web対応
